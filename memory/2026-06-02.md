### 22:30  - WebScanner项目分析与答辩PPT生成

**任务**：分析 D:\IDEA\Project\WebScanner Java 项目源码，生成答辩 PPT

**项目概要**：
- 项目名称：WebScanner（Web 漏洞扫描系统）
- 技术栈：Java 8 + Spring MVC 5.1.9 + MyBatis 3.5.2 + jsoup 1.17.2 + MySQL 8.0.30 + HikariCP
- 架构：三层架构（表示层 JSP / 控制层 Controller / 业务层 Service / 持久层 DAO）
- 核心模块：
  - SQL 注入检测（SecurityDetector.SQLInjectionDetector）
  - XSS 检测（SecurityDetector.XSSDetector）
  - URL 扫描（MyUrl.scan()）
  - HTML 解析（HtmlParser，基于 jsoup，DOM + CSS 双模式）
  - 用户认证（UserService + AuthFilter）
- 扫描原理：基准请求 → Payload 注入 → 响应差异分析（状态码/响应时间/内容长度/SQL报错）

**PPT 生成**：
- 使用 python-pptx 库（本次会话中安装：pip install python-pptx）
- 脚本路径：C:\Users\hp\Desktop\generate_webscanner_ppt.py
- 输出路径：C:\Users\hp\Desktop\WebScanner_项目答辩.pptx
- 共 16 页：封面、目录、背景意义、技术栈、架构、模块、SQL注入详解、XSS详解、URL扫描详解、REST API接口、SQL自动扫描代码、MyBatis安全写法、项目难点、功能演示、总结展望、致谢Q&A
- 设计风格：科技蓝（#1A73E8）+ 深色代码块 + 左文右代码布局

**问题记录**：
- Windows GBK 编码（CP936）无法处理 emoji，print 语句中的 emoji 会导致 UnicodeEncodeError
- 解决方法：移除 print 中的 emoji，改用纯 ASCII 字符
- pptx skill 的 SKILL.md 符号链接指向错误（指向了 find-skills），需修正

**关键发现**：
- WebScanner 项目 UserServiceImpl 中同时实现了漏洞 SQL（字符串拼接）和安全 SQL（PreparedStatement）对比演示
- MyBatis Mapper XML 中 ${} 是字符串替换（危险），#{} 是参数化查询（安全）
- AuthFilter 精确区分了 JSP 页面拦截和 /security/* API 放行
